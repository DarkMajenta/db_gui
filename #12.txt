Для представления информационной системы поликлиники в формате UML, вот диаграмма сущность-связь, которая соответствует описанным сущностям и их атрибутам:

```
  +----------------+         +--------------+         +-----------+
  |    Врачи       |         |   Пациенты   |         | Отделения |
  +----------------+         +--------------+         +-----------+
  | - Врач_ID      |         | - Пациент_ID |         | - Отделение_ID |
  | - Фамилия      |         | - Фамилия    |         | - Название     |
  | - Имя          |         | - Имя        |         | - Этаж         |
  | - Отчество     |         | - Отчество   |         | - Номера_кабинетов |
  | - Должность    |         | - Адрес      |         | - ФИО_заведующего |
  | - Стаж_работы  |         | - Город      |         +-----------+
  | - Научное_звание|         | - Возраст    |
  | - Адрес        |         | - Пол        |
  | - Номер_отделения|
  +----------------+
         |
         |
  +-----------------+
  |    Диагнозы     |
  +-----------------+
  | - Диагноз_ID    |
  | - Название      |
  | - Признаки_болезни |
  | - Период_лечения |
  | - Назначения    |
  +-----------------+
         |
         |
  +-------------------+
  | Истории болезней  |
  +-------------------+
  | - История_ID     |
  | - Пациент_ID     |
  | - Врач_ID        |
  | - Диагноз_ID     |
  | - Лечение        |
  | - Дата_заболевания |
  | - Дата_излечения |
  | - Вид_лечения
  +-------------------+

```

- `Врачи`, `Пациенты`, `Отделения` - основные сущности предметной области.
- Идентифицирующие атрибуты:
  - `Врач_ID` - идентификатор врача в сущности "Врачи".
  - `Пациент_ID` - идентификатор пациента в сущности "Пациенты".
  - `Отделение_ID` - идентификатор отделения в сущности "Отделения".
  - `Диагноз_ID` - идентификатор диагноза в сущности "Диагнозы".
  - `История_ID` - идентификатор истории болезни в сущности "Истории болезней".
- Обязательные атрибуты:
  - В сущности "Врачи": Фамилия, Имя, Отчество, Должность, Стаж_работы.
  - В сущности "Пациенты": Фамилия, Имя, Отчество, Адрес, Город, Возраст, Пол.
  - В сущности "Отделения": Название, Этаж, Номера_кабинетов, ФИО_заведующего.
  - В сущности "Диагнозы": Название, Признаки_болезни, Период_лечения, Назначения.
  - В сущности "Истории болезней": Пациент_ID, Врач_ID, Диагноз_ID, Лечение, Дата_заболевания, Дата_излечения, Вид_лечения.
- Отношения между сущностями:
  - Врачи и Пациенты: Ассоциация ("Врачи" связаны с "Пациенты").
  - Врачи и Отделения: Ассоциация ("Врачи" связаны с "Отделения").
  - Лечение: Ассоциация ("Врачи" связаны с "Пациенты" через "Истории болезней").
  - Врачи и Диагнозы: Ассоциация ("Врачи" связаны с "Диагнозы").
  - Пациенты и Диагнозы: Ассоциация ("Пациенты" связаны с "Диагнозы").
  - История болезни: Ассоциация ("Пациенты" и "Врачи" связаны с "Истории болезней", "Диагнозы" связан с "Истории болезней").

UML:


@startuml

!define ENTITY(name, fields) entity name as "name" << (E, #FFAAAA) >> {
  fields
}

!define RELATIONSHIP(from, to, type) from --{type} to

skinparam ArrowColor #000000
skinparam ArrowThickness 1
skinparam ArrowFontColor #000000
skinparam ArrowFontName Courier
skinparam EntityBackgroundColor #FFFFDD
skinparam EntityBorderColor #000000
skinparam EntityFontSize 14
skinparam EntityFontName Courier
skinparam EntityFontColor #000000

ENTITY "Врачи" as en1 << (E, #FFAAAA) >> {
  <Врач_ID>
  Фамилия
  Имя
  Отчество
  Должность
  Стаж_работы
}

ENTITY "Пациенты" as en2 << (E, #FFAAAA) >> {
  <Пациент_ID>
  Фамилия
  Имя
  Отчество
  Адрес
  Город
  Возраст
  Пол
}

ENTITY "Отделения" as en3 << (E, #FFAAAA) >> {
  <Отделение_ID>
  Название
  Этаж
  Номера_кабинетов
  ФИО_заведующего
}

ENTITY "Диагнозы" as en4 << (E, #FFAAAA) >> {
  <Диагноз_ID>
  Название
  Признаки_болезни
  Период_лечения
  Назначения
}

ENTITY "Истории болезней" as en5 << (E, #FFAAAA) >> {
  <История_ID>
  Пациент_ID
  Врач_ID
  Диагноз_ID
  Лечение
  Дата_заболевания
  Дата_излечения
  Вид_лечения
}

RELATIONSHIP(en1, en2, "-")
RELATIONSHIP(en1, en3, "-")
RELATIONSHIP(en1, en4, "-")
RELATIONSHIP(en2, en4, "-")
RELATIONSHIP(en1, en5, "-")
RELATIONSHIP(en2, en5, "-")
RELATIONSHIP(en4, en5, "-")

@enduml
